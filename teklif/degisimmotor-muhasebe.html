Teknik Zorluklar

Zor alanları biliyoruz,
çözüm yollarını da.

Yüksek

OCR Vade Tespiti

Neden Zor

Fatura formatı standart değil — vade "90 gün", "NET 90", sadece tarih ya da hiç yazılmıyor. Taranmış kağıt PDF'lerde OCR doğruluğu düşük.

Nasıl Çözeriz

E-fatura/e-arşiv kullanan tedarikçilerde OCR'a gerek yok — XML'den yapısal okuma. OCR sadece eski dijital olmayan belgeler için. Düşük güvende muhasebeciye bırakılır.

Orta-Yüksek

AI Mail Analizi

Neden Zor

Müşteri yanıtları belirsiz olabilir ("yakında öderiz"). Türkçe iş dili nüansları — taahhüt mü, nezaket mi? Birden fazla konu tek mailde.

Nasıl Çözeriz

Güven skoru eşiği: %70 altı → muhasebeciye bırakılır. İlk aylar eğitim verisi toplanır, model sürekli iyileşir.

Orta

Logo API Kapsamı

Neden Zor

API dokümantasyonu eksik/eski olabilir. İhtiyaç duyulan bazı alanlar API'dan gelmeyebilir.

Nasıl Çözeriz

Faz 0 discovery'de API tam keşfedilir. Eksik alan varsa workaround veya scope ayarı yapılır.

Orta

3-Yönlü Eşleme

Neden Zor

Ürün kodu/ismi sipariş ↔ irsaliye ↔ faturada farklı olabilir. Kısmi teslimatlar, iskonto farkları.

Nasıl Çözeriz

Fuzzy matching + kalem kodu standardizasyonu. Eşleşemeyen → muhasebeciye yönlendirilir.

Düşük-Orta

Kredi Kartı Excel

Neden Zor

10 farklı banka = 10 farklı Excel formatı. Sütun isimleri, tarih formatları, tutar gösterimi farklı.

Nasıl Çözeriz

Banka başına şablon bir kez ayarlanır. Yeni format → kısa sürede şablon eklenir.

Düşük

Mail Teslim

Neden Zor

Toplu otomatik mailler spam'e düşebilir.

Nasıl Çözeriz

SPF/DKIM/DMARC ayarı; ilk hafta test gönderim; müşteri beyaz liste.
